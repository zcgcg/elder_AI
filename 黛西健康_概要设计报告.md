## 2. 系统概述

### 2.1 系统目标

构建一套面向养老健康服务领域的综合管理平台，实现用户健康管理、服务人员调度、商品运营、交易处理、数据分析的一体化后台管理。核心目标包括：

- **用户管理**：统一管理用户档案、健康数据、用药记录、资产信息
- **服务管理**：全流程管控服务人员、审核、工单派发与执行
- **商品运营**：多品类健康服务商品（家政护理/康复理疗/上门体检）的上架与售卖
- **内容运营**：生活圈动态、健康资讯、健康讲堂、活动管理等内容生态运营
- **交易管理**：订单全生命周期管理（下单→派单→服务→售后）
- **数据分析**：用户、交易、服务、评价等维度的数据可视化分析
- **系统管理**：员工权限、角色配置、审计日志等安全管控

### 2.2 用户角色

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| 超级管理员 | 系统最高权限者 | 全部模块、全部操作 |

## 3. 总体架构设计

### 3.1 系统架构

本系统采用**前后端分离 JavaEE 单体架构**，前端 SPA 通过 HTTP/HTTPS 请求与后端 RESTful API 交互，后端以分层方式组织代码，打包为单一可部署的 Jar 包运行。

```
┌──────────────────────────────────────────────────────────┐
│                    前端展示层 (SPA)                        │
│   Vue 3 + Element Plus + Pinia + ECharts                 │
│   Axios HTTP 请求，Hash 路由                               │
├──────────────────────────────────────────────────────────┤
│                     Nginx (反向代理 + 静态资源)              │
├──────────────────────────────────────────────────────────┤
│                    后端服务层 (Spring Boot)                │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  Controller 层 (RESTful API，Knife4j 文档)            │ │
│  ├─────────────────────────────────────────────────────┤ │
│  │  Service 层 (业务逻辑)                                │ │
│  │  用户管理 | 服务管理 | 商品管理 | 运营管理               │ │
│  │  交易管理 | 数据分析 | 系统设置 | 通知服务               │ │
│  ├─────────────────────────────────────────────────────┤ │
│  │  Mapper 层 (MyBatis 数据访问)                         │ │
│  └─────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────┤
│                    数据存储层                              │
│   MySQL 8.0 (业务数据) + Redis (缓存/会话) + OSS (文件)     │
└──────────────────────────────────────────────────────────┘
```

**分层说明：**

| 层次 | 职责 | 技术 |
|------|------|------|
| Controller | 接收 HTTP 请求，参数校验，调用 Service，返回 JSON | Spring MVC |
| Service | 核心业务逻辑，事务管理 | Spring + 业务 POJO |
| Mapper | 数据库 CRUD 操作 | MyBatis |
| Entity | 数据库表映射实体类 | POJO |
| Common | 通用工具、拦截器、异常处理、配置 | Spring AOP / Interceptor |

### 3.2 技术选型

| 层次 | 技术方案 | 版本 | 说明 |
|------|----------|------|------|
| JDK | JDK 8 | 1.8 | LTS 长期支持版本，生态成熟 |
| 前端框架 | Vue 3 | 3.x | 渐进式前端框架，Composition API |
| UI 组件库 | Element Plus | 2.x | 与原型左侧导航+表单+表格+图表风格匹配 |
| 状态管理 | Pinia | 2.x | Vue 3 官方推荐状态管理 |
| HTTP 客户端 | Axios | 1.x | 前端请求拦截、统一错误处理 |
| 图表可视化 | ECharts | 5.x | 仪表盘、数据分析模块图表渲染 |
| 后端框架 | Spring Boot | 2.7.18 | 适配 JDK 8 的最高版本，JavaEE 单体架构，内嵌 Tomcat |
| ORM 框架 | MyBatis | 3.x | SQL 与 Java 对象映射，灵活控制 SQL |
| API 文档 | Knife4j（基于 Swagger） | 3.x | 基于 Swagger 规范，提供更优的在线文档 UI 与调试体验，兼容 JDK 8 + Spring Boot 2.x |
| 数据库 | MySQL | 8.0 | 用户/服务/商品/交易/运营全量业务数据 |
| 缓存 | Redis | 7.x | 会话管理、验证码、热点数据缓存 |
| 文件存储 | 阿里云 OSS | — | 头像、报告、视频、轮播图等文件 |
| Web 服务器 | Nginx | 1.x | 静态资源服务、反向代理、负载均衡 |
| 部署平台 | 阿里云 ECS | — | 云服务器，CentOS 7 / Alibaba Cloud Linux |

### 3.3 部署架构

系统部署于**阿里云**平台，采用一台或多台 ECS 云服务器承载 Spring Boot 单体应用，通过 Nginx 反向代理分发流量，关键依赖服务采用云产品：

```
                     ┌──────────────────┐
                     │  用户浏览器 (HTTPS) │
                     └────────┬─────────┘
                              │
                     ┌────────┴─────────┐
                     │  阿里云 SLB       │
                     │  (负载均衡, 可选)  │
                     └────────┬─────────┘
                              │
                     ┌────────┴─────────┐
                     │  Nginx (ECS)      │
                     │  Vue 静态资源 +    │
                     │  反向代理 /api/*   │
                     └────────┬─────────┘
                              │
              ┌───────────────┼───────────────┐
              │               │               │
         ┌────┴────┐    ┌────┴────┐     ┌────┴────┐
         │ Spring   │    │ Spring   │     │ Spring   │
         │ Boot 实例 │    │ Boot 实例 │     │ Boot 实例│
         │  (ECS)   │    │  (ECS)   │     │  (ECS)   │
         │ Jar 包运行│    │ Jar 包运行│     │ Jar 包运行│
         └────┬────┘    └────┬────┘     └────┬────┘
              │              │                │
              └──────────────┼────────────────┘
                             │
              ┌──────────────┼────────────────┐
              │              │                │
         ┌────┴────┐   ┌────┴────┐      ┌────┴────┐
         │ 阿里云    │   │ 阿里云    │      │ 阿里云    │
         │ RDS MySQL │   │ Redis    │      │ OSS      │
         │ 主从架构   │   │ 缓存服务  │      │ 对象存储  │
         └──────────┘   └─────────┘      └─────────┘
```

**部署说明：**
- **ECS**：部署 Spring Boot 应用 Jar 包及 Nginx，由 SLB（可选）或 Nginx 负载均衡至多实例
- **RDS MySQL**：托管数据库，自带主从架构与自动备份
- **Redis**：托管缓存服务，免运维，支持主备切换
- **OSS**：文件存储，头像/报告/视频/轮播图等静态资源
