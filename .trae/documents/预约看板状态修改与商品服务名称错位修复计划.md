# 预约看板状态修改与商品服务名称错位修复计划

## 一、问题概述

### 问题1：预约看板缺少状态修改功能
当前预约看板页面只能查看预约状态（待服务、服务中、已完成、已取消），但无法修改状态。管理员需要能够将预约状态从"待服务"改为"服务中"或"已完成"。

### 问题2：商品服务管理编辑时名称标签错位
在商品服务管理的编辑弹窗中，"商品服务名称"标签因宽度不足导致"称"字换行到第二行，显示不美观。

---

## 二、当前状态分析

### 2.1 预约看板状态修改

**前端现状（ScheduleView.vue）：**
- 预约卡片底部显示状态标签 + 删除按钮，无状态修改入口
- 状态映射：pending→待服务, service_in→服务中, completed→已完成, cancelled→已取消
- API 调用：`getAppointments()` 获取列表、`createAppointment()` 创建、`deleteAppointment()` 删除
- 缺少：更新预约状态的 API 方法

**后端现状：**
- `DashboardController.java` 只提供了 `GET /appointments`、`POST /appointments`、`DELETE /appointments/{id}` 三个接口
- 缺少 `PUT /appointments/{id}` 更新接口
- 但 `JdbcAdminDataService.updateResource()` 方法已支持 "appointments" 类型的状态更新（第561-573行），状态通过 `workOrderStatus()` 方法转换中文状态码

**数据层（JdbcAdminDataService.java 第561-573行）：**
```java
} else if ("workOrders".equals(name) || "appointments".equals(name)) {
    ...
    if (payload.containsKey("status")) values.put("status", workOrderStatus(text(payload, "status", "待服务")));
    ...
    updateById("work_order", id, values);
}
```
后端数据层已支持状态更新，只需在 Controller 层暴露接口。

### 2.2 商品服务名称标签错位

**现状分析：**
- 位于 `GenericListView.vue` 第42行：`<el-form :model="form" label-width="96px">`
- 商品服务名称字段（第264行）：`{ prop: 'name', label: '商品服务名称', ... }`
- `label-width="96px"` 对于6个汉字的标签来说宽度不足，导致文字自动换行
- Element Plus 默认 `el-form-item__label` 右对齐，宽度不够时自动折行

---

## 三、修复方案

### 3.1 预约看板状态修改功能

#### 后端修改（1个文件）

**文件：`daisy-health-admin-backend/src/main/java/com/daisy/health/controller/DashboardController.java**

修改内容：
- 新增 `PUT /api/v1/appointments/{id}` 接口
- 调用 `dataService.updateResource("appointments", id, payload)` 更新预约
- 返回更新后的数据

#### 前端 API 层修改（1个文件）

**文件：`daisy-health-admin-frontend/src/api/http.js`**

修改内容：
- 新增 `updateAppointment(id, payload)` 函数，调用 `PUT /appointments/${id}`

#### 前端页面修改（1个文件）

**文件：`daisy-health-admin-frontend/src/views/ScheduleView.vue`**

修改内容：
1. 在预约卡片的 `.appointment-card-footer` 中，将状态标签改为可点击的下拉选择器（`el-select` 小型化）
2. 或在状态标签旁添加一个"修改状态"按钮，点击弹出状态选择下拉菜单
3. 实现状态切换时调用更新 API，成功后刷新列表
4. 状态选项：待服务、服务中、已完成、已取消（与工单管理一致）

**推荐方案：使用 el-dropdown 下拉菜单**
- 点击状态标签或旁边的小箭头，弹出下拉菜单选择目标状态
- 视觉上保持卡片简洁，操作直观

### 3.2 商品服务名称标签错位修复

**文件：`daisy-health-admin-frontend/src/views/GenericListView.vue`**

修改内容：
- 将 el-form 的 `label-width` 从 `96px` 调整为 `112px`
- 这样可以容纳6个汉字的标签而不换行
- 同时检查其他页面的表单 label-width 是否也需要调整

**验证范围：**
- 商品服务管理（6字标签：商品服务名称）
- 其他使用 GenericListView 的所有模块

## 四、具体实施步骤

### 步骤1：后端添加预约更新接口
- 在 DashboardController.java 中添加 `@PutMapping("/appointments/{id}")` 方法

### 步骤2：前端添加更新 API
- 在 http.js 中添加 `updateAppointment` 导出函数

### 步骤3：预约看板添加状态修改功能
- 在 ScheduleView.vue 的预约卡片中添加状态下拉选择
- 实现状态变更的调用逻辑和成功提示

### 步骤4：修复商品服务名称标签错位
- 修改 GenericListView.vue 中表单的 label-width

## 五、验证步骤

### 预约看板状态修改验证：
1. 打开预约看板页面
2. 确认每个预约卡片都有状态修改入口
3. 将"待服务"改为"服务中"，确认状态标签更新且后端数据同步
4. 将"服务中"改为"已完成"，确认更新成功
5. 刷新页面，确认状态持久化

### 商品服务名称标签验证：
1. 打开商品服务管理页面
2. 点击"新增"或"编辑"按钮
3. 确认"商品服务名称"标签完整显示在一行，不换行
4. 检查其他模块的编辑弹窗，确认标签显示正常
